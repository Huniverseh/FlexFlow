### **健身计划网页应用开发提示词**

#### **1. 项目核心概述**

我需要你为我创建一个名为“FlexFlow”（或者其他你认为合适的名字）的轻量级网页健身辅助应用（Web App）。

**核心目标：** 让用户可以创建、管理和执行自定义的健身训练计划。应用的核心是**自动化组间休息计时**和**清晰的训练流程引导**，将用户从频繁看表和记忆流程的干扰中解放出来。同时，应用需具备通过**链接一键分享和导入**训练计划的社交功能。

**设计理念：** 简洁、专注、高效。界面设计应清晰直观，避免任何不必要的功能分散用户注意力。应用应为响应式设计，在桌面和移动端浏览器上都有良好的体验。

#### **2. 技术栈与数据存储**

*   **前端框架：** 使用现代 JavaScript 框架，如 **React** 或 **Vue**，来构建一个单页面应用（SPA）。
*   **数据存储：** 为实现轻量化和免后端部署，所有用户数据（如训练计划、动作库、个人信息等）均存储在浏览器的 **Local Storage** 中。

#### **3. 核心数据模型**

请基于以下数据结构进行开发：

*   **`Action` (动作):**
    *   `id`: 唯一标识符 (e.g., UUID)
    *   `name`: 动作名称 (e.g., "杠铃卧推")
    *   `targetPart`: 目标部位 (e.g., "胸部")
    *   `imageURL`: 动作演示图的 URL (可选，若无则显示默认占位图)
*   **`WorkoutPlan` (训练计划):**
    *   `id`: 唯一标识符
    *   `name`: 计划名称 (e.g., "周一胸部强化训练")
    *   `actions`: 一个包含具体训练步骤的数组，数组中的每个对象格式为：
        *   `actionId`: 关联 `Action` 的 `id`
        *   `weight`: 重量 (e.g., "50kg")
        *   `reps`: 次数 (e.g., "12次")
        *   `sets`: 组数 (e.g., 4)
        *   `restSeconds`: 组间休息时间 (e.g., 60)
*   **`User` (用户个人信息):**
    *   `height`: 身高 (cm)
    *   `weight`: 体重 (kg)
    *   `bodyFat`: 体脂率 (%)
*   **`WorkoutRecord` (训练记录):**
    *   `id`: 唯一标识符
    *   `planId`: 完成的训练计划 `id`
    *   `planName`: 完成的训练计划名称（冗余存储，便于展示）
    *   `date`: 完成日期 (e.g., "YYYY-MM-DD")

#### **4. 页面与功能模块详细拆解**

应用由一个带底部导航栏的主体框架和几个核心页面组成。

**模块一：主页 (Home Page)**

*   **布局：** 页面顶部有应用标题和两个操作按钮。主体内容为训练计划卡片列表。
*   **顶部操作区：**
    *   **“+”号按钮：** 点击后弹出一个菜单，包含两个选项：“创建新计划” (跳转到计划编辑页) 和 “添加新动作” (打开一个用于创建 `Action` 的弹窗)。
    *   **“...”号按钮：** 点击后提供“从链接导入”功能，弹出一个输入框让用户粘贴分享链接。
*   **训练计划卡片 (`WorkoutPlanCard`)：**
    *   以列表形式垂直排列。
    *   每张卡片清晰展示 `WorkoutPlan` 的 `name`、包含的动作总数、预估总训练时长。
    *   卡片上有一个**“分享”按钮**。
    *   **交互：** 点击整个卡片，会弹出一个**计划详情模态框(Modal)**，展示该计划包含的所有动作、组数、次数等详细信息。模态框底部有一个醒目的**“开始训练”**按钮，点击后跳转到“训练中”页面。

**模块二：计划创建/编辑页 (Plan Editor Page)**

*   **用途：** 创建新的 `WorkoutPlan` 或编辑已有的计划。
*   **布局：**
    *   顶部是计划名称的输入框。
    *   主体是“已选动作列表”，用户可以在此区域通过**拖拽**调整动作顺序。
    *   列表中的每一个动作项，都应有关联的输入框，用于设置其 `weight`, `reps`, `sets`, `restSeconds`。
    *   页面底部或侧边有一个“从动作库添加”的按钮。
*   **交互流程：**
    1.  点击“从动作库添加”，弹出一个包含所有已创建 `Action` 的列表。
    2.  用户可以勾选一个或多个动作添加至当前计划。
    3.  返回编辑页后，为每个动作配置具体参数。
    4.  通过拖拽调整顺序。
    5.  点击“保存计划”，将该 `WorkoutPlan` 对象存入 Local Storage 并返回主页。

**模块三：训练中页面 (Live Training Page)**

*   **设计：** 这是一个高度专注的“执行”界面。字体要大，信息要醒目，对比度要高。
*   **信息展示区：**
    *   当前动作名称 (`Action.name`) 及演示图。
    *   当前组数进度 (e.g., "第 2 / 4 组")。
    *   下一个动作的名称（灰色小字提示）。
*   **核心交互区（页面中央）：**
    *   **状态一 (执行中):** 显示一个巨大的**“完成本组”**按钮。
    *   **状态二 (休息中):** “完成本组”按钮变为一个巨大的**圆形倒计时器**，清晰显示剩余休息秒数。
*   **工作流程：**
    1.  进入页面，显示第一个动作的第一组信息，进入“执行中”状态。
    2.  用户完成动作后，点击“完成本组”按钮。
    3.  应用**立刻播放一声提示音**，并切换到“休息中”状态，倒计时开始。
    4.  倒计时归零时，**再次播放一声提示音**，并弹出一个**确认模态框**（例如：“准备好！下一组：卧推”），需要用户手动点击“开始”确认。
    5.  点击确认后，模态框消失，界面更新为下一组的训练信息，返回“执行中”状态。
    6.  当一个计划所有动作全部完成后，跳转到一个“恭喜完成训练！”的祝贺页面，并自动将一条 `WorkoutRecord` 存入 Local Storage。该页面有一个“返回主页”的按钮。

**模块四：分享与导入机制**

*   **分享：** 点击计划卡片上的“分享”按钮，执行以下操作：
    1.  获取该 `WorkoutPlan` 的完整 JSON 对象。
    2.  将该 JSON 对象进行 **Base64** 或类似的编码，使其变为一个安全的字符串。
    3.  生成一个包含该字符串的 URL (e.g., `https://your-app-domain.com/#/import?data=ENCODED_STRING`)。
    4.  将该 URL 复制到用户剪贴板，并提示“已复制分享链接”。
*   **导入：** 用户在主页点击“导入”并粘贴上述链接后，执行以下操作：
    1.  从 URL 中解析出 `data` 参数。
    2.  对该字符串进行解码，还原为 `WorkoutPlan` 的 JSON 对象。
    3.  为其生成一个新的 `id`，然后将其存入 Local Storage。
    4.  提示“导入成功！”，并刷新主页列表。

**模块五：底部导航栏 (Bottom Navigation)**

*   固定在页面底部，包含三个图标链接：
    *   **主页 (Home):** 导航至主页。
    *   **记录 (Records):** 导航至训练记录页。该页面以**日历**形式展示，所有存在 `WorkoutRecord` 的日期会被高亮标记。点击高亮日期，可以查看当天完成的训练计划详情。
    *   **我的 (Profile):** 导航至个人中心页。该页面提供简单的表单，用于让用户输入和更新自己的 `height`, `weight`, `bodyFat` 等信息。
